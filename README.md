# cinemabooking
Техническое задание на разработку системы бронирования билетов в кино

**1. Цель проекта**
Разработать веб-приложение для бронирования билетов в кино, позволяющее пользователям просматривать расписание сеансов, выбирать места в зале и бронировать билеты.

**2. Функциональные требования**

**2.1. Аутентификация**
Пользователи должны иметь возможность регистрироваться и входить в систему, используя имя пользователя и пароль.
Должна быть реализована двухфакторная аутентификация через Telegram.

**2.2. Роли пользователей**
Посетитель - может просматривать расписание, выбирать сеансы, бронировать билеты.
Администратор - может добавлять/редактировать/удалять сеансы, просматривать список забронированных билетов, управлять учетными записями пользователей.

**2.3. Расписание и бронирование**
Пользователи могут просмотреть текущее расписание сеансов.
Пользователи могут выбрать сеанс и просмотреть доступные места.
Пользователи могут бронировать билеты на выбранное место.

**2.4. Интеграция с внешними сервисами**
Интеграция с Telegram для реализации чат-бота, который будет уведомлять пользователей о статусе их бронирования.

**3. Технические требования**
Клиентская часть должна быть реализована на JavaScript, с использованием технологии AJAX для асинхронной загрузки данных.
Серверная часть должна быть реализована на PHP.
Можно использовать стандартные библиотеки, такие как jQuery, и фреймворки, такие как Angular или React.

**4. Этапы разработки**
Проектирование архитектуры приложения.
Разработка серверной части на PHP.
Разработка клиентской части на JavaScript.
Интеграция с внешними сервисами.
Тестирование приложения.
Деплой и поддержка приложения.
**5. Сроки**
Сроки разработки и внедрения системы определяются в процессе уточнения требований и планирования работы над проектом.
